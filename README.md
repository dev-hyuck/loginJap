개발 전, 공통조건

● 모든 테이블은 고유 식별자 ID를 가진다.

● 3 Layer Architecture에 따라 각 Layer의 목적에 맞게 개발하기

● CRUD 필수 기능은 모두 데이터베이스 연결 및 JPA를 사용해야 된다.

●  JPA 연관관계는 단방향만  적용해라!

● 인증/ 인가 절차는 Cookie/Session을 활용하여 개발하기

 
Lv 1. 일정 CRUD

작성 유저명, 할일 제목, 할일 내용, 작성일, 수정일 필드

작성일 , 수정일 필드는 JAP Auditing을 활용한다.

 

Lv 2. 유저 CRUD

유저명, 이메일, 작성일, 수정일 필드

작성일 , 수정일 필드는 JAP Auditing을 활용한다. 

 

연관관계 구현

-> 일정은 이제 작성 유저명 필드 대신 유저 고유 식별자 필드를 가집니다.

 

Lv 3. 회원가입

유저에 비밀번호 필드를 추가

● 비밀번호는 8글자 이상

● 비밀번호 암호화는 도전 기능

 

Lv 4. 로그인 (인증) LV 1 ~4 (필수 )

설명

-> cookie/Session을 활용해 로그인 기능을 구현합니다.

조건

-> 이메일과 비밀번호를 활용해 로그인 기능을 구현합니다.

-> 필요한 API들에서 세션을 활용합니다.
